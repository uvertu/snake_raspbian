# snake\_raspbian

Хохряков Антон Сергеевич - uvertu
Чиртулов Матвей Вениаминович - MatveyChirtulov



Проект представляет собой игру "Змейка" с серверной частью на Python (Pygame) и удаленным управлением через Android-приложение. Сервер отображает игровое поле на компьютере, а мобильное устройство используется как контроллер для управления змейкой по сети.



Основные функции

Серверная часть (snake\_game.py):

* Полноценная игра "Змейка" с графическим интерфейсом Pygame
* Сетевой сервер для подключения Android-устройств
* Отображение игрового состояния в реальном времени
* Локальное управление (для тестирования)
* Автоматическое определение IP-адреса для подключения



Android-приложение:

* Удаленное управление змейкой (вверх/вниз/влево/вправо)
* Запуск новой игры, пауза, завершение
* Отображение текущего счета и статуса игры
* Автоматическое переподключение при разрыве соединения



Технические требования

Для серверной части:

* Python 3.7+
* Pygame 2.0+
* Работа в локальной сети с поддержкой TCP/IP



Для Android-приложения:

* Android 5.0 (API 21) или выше
* Разрешение на использование интернета
* Подключение к той же локальной сети, что и сервер



Установка и запуск

1\. Установка серверной части



pip install pygame

python snake\_game.py



После запуска сервер автоматически определит локальный IP-адрес и выведет его в консоль.



2\. Настройка Android-приложения



Использование готового APK-файла

(snake\_control\_app/app/release/app-release.apk)



3\. Подключение устройств

* Запустите сервер на компьютере
* Запишите IP-адрес, который отобразится в консоли



На Android-устройстве:

* Введите IP-адрес сервера в поле ввода
* Нажмите "ПОДКЛЮЧИТЬСЯ"



При успешном подключении интерфейс изменится



Управление игрой



С Android-устройства:



* НОВАЯ ИГРА - начать новую игру
* ПАУЗА - приостановить/возобновить игру
* Стрелочки - управление направлением змейки
* ЗАВЕРШИТЬ - закончить текущую игру



Сетевая архитектура:



Android Device (Client) ↔ TCP/IP ↔ Python Server (Game Host)

&nbsp;   ↓                                  ↓

Управление                      Отображение игры

Статус игры                     Логика игры

&nbsp;                               Отправка состояния

* Порт: 5555 (по умолчанию)
* Протокол: TCP
* Формат данных: JSON
